실습 소개

모니터링에는 두 가지 목표가 있습니다 - 즉, 상황에 대한 인식을 실현하여 시기적절하고 효과적인 대응을 제공하는 것과 사전 예방적 행동 과정을 가능하게 하는 비즈니스, 개발 및 운영에 대한 통찰력을 얻는 것입니다.

이 워크샵에서는 이러한 목표를 달성하기 위한 워크로드 모니터링 전략을 개발하고 구현하는 프로세스를 안내합니다. 다음 방법을 배우게 됩니다.

  1. 모니터링 계획 수립
  2. AWS Services를 사용하여 계획 구현
    1. AWS Systems Manager Run Command 및 Parameter Store를 사용하여 원격 분석 소스 구성
    2. Amazon CloudWatch Logs를 사용하여 원격 분석 수집
    3. Amazon CloudWatch Log-Filter 지표를 사용하여 지표 및 임계 값 설정
    4. Amazon CloudWatch 경보를 사용하여 지표 임계 값을 초과 할 때 발생하도록 경보 구성
    5. Amazon Simple Notification Service를 사용하여 알림 주제 및 구독 설정
    6. AWS Lambda를 사용하여 자동 응답 구성

이 실습에서는 주요 이벤트 범주에 대해 경고하고, 운영 결과를 모니터링하고, 응답을 트리거하며, 통찰력을 제공하는 방법을 보여줍니다.

실습페이지와 실제 콘솔에서 보는 이미지에 차이가 있을 수 있습니다. 이 점 참고부탁드립니다.
