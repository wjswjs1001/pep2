실습1. CloudWatch Handson Lab

실습 환경 구성

이 실습을 수행하려면 다음 사항이 필요합니다.

  • 실습은 관리자 권한과 기존 EC2 키 페어가있는 사용자로 로그인하여 수행해야합니다.
  • us-east-1 region에서 사용 가능한 VPC가 필요합니다.

1.1 CloudFormation을 사용하여 실습 환경 배포

CloudFormation 스택을 시작하는 데는 두 가지 옵션이 있습니다. 첫 번째 옵션은 CloudFormation 스택 변수를 자동으로 채웁니다. 두 번째 옵션은 템플릿 URL에서 새 스택을 수동으로 생성하는 것과 관련된 모든 단계를 안내합니다.

Option A: 스택 생성 변수 자동 실행 및 채우기

여기를 클릭하여 Lab 배포

NOTE: 자동 시작 프로세스는 새 버전의 CloudFormation 콘솔을 사용하는 경우에만 작동합니다.

버튼이 새 스택 생성 페이지로 이동하지 못하면 CloudFormation 드롭 다운 메뉴에서 새 콘솔 링크를 클릭하여 새 CloudFormation 콘솔로 전환해보십시오.

Option B: 템플릿 URL에서 스택 생성을 수동으로 안내

  1. CloudFormation 스크립트 URL 복사 for this lab: distributed.yaml

NOTE: 아래 지침은 CloudFormation 콘솔의 새 버전을 기반으로 합니다. 버튼을 눌러 새 스택 생성 페이지로 이동하지 못하면 CloudFormation 드롭 다운 메뉴에서 새 콘솔 링크를 클릭하여 새 CloudFormation 콘솔로 전환해보십시오.

  1. 관리자 계정을 사용하여 CloudFormation 콘솔에 액세스하십시오. https://console.aws.amazon.com/cloudformation/.
  2. Create Stack 버튼을 클릭하십시오.
  • Specify Template step에서:
  • Prerequisite - Prepare template 섹션에서 Template is ready 라디오 버튼이 선택되어 있는지 확인합니다.
  • Specify template 섹션에서 Amazon S3 URL 라디오 버튼이 선택되어 있는지 확인한 다음 복사된 URL을 Amazon S3 URL 텍스트 상자에 붙여 넣습니다.

NOTE: CloudFormation 템플릿은 optional and required sections을 모두 포함하는 AWS 인프라를 설명하는 JSON 또는 YAML 형식의 텍스트 파일입니다. 다음 단계에서는 구현될 리소스를 정의하는데 도움이 되도록 템플릿에 전달될 스택 및 매개 변수의 이름을 제공합니다.

  1. Specify Template 단계에서, Amazon S3 템플릿 URL 지정 옆에있는 Designer에서 View 링크를 클릭합니다.

  • 잠시 시간을내어 생성하려는 환경의 그래픽 표현과 JSON 및 YAML 형식의 템플릿을 살펴보십시오.

  • 템플릿은 중첩 스택으로 YAML로 작성되었으며 원하는 경우 JSON으로 변환할 수도 있습니다.

    NOTE: AWS CloudFormation Designer는 AWS CloudFormation 템플릿을 생성,보기 및 수정하기 위한 그래픽 도구입니다. Designer를 사용하면 끌어서 놓기 인터페이스를 사용하여 템플릿 리소스를 다이어그램으로 표시한 다음 통합된 JSON 및 YAML 편집기를 사용하여 세부 정보를 편집할 수 있습니다. AWS CloudFormation Designer는 템플릿 리소스 간의 상호 관계를 빠르게 확인하고 템플릿을 쉽게 수정할 수 있도록 도와줍니다.

  1. 스택 생성 아이콘 (a cloud with an arrow) 을 선택하여 템플릿 지정 단계로 돌아갑니다.

    NOTE: Designer에서 Create Stack 옵션을 클릭하면 선택한 리전에 새 cf-template S3 버킷이 자동으로 생성되고 원래 템플릿이 고유한 임의 이름으로 복사되어 새 S3 버킷에 배치됩니다.

  2. 템플릿 지정 단계에서 다음 버튼을 클릭합니다.

  3. 세부 정보 지정 페이지에서:

  • 스택 이름 정의 : CWLab을 입력
  • 이메일 주소를 입력하십시오. 이것은 실습에서만 사용되며 이벤트에 대한 SNS 알림을 보내는데만 사용됩니다.
  • 리소스에 연결할 수 있는 사용자를 제한하려면 HTTPLocation CIDR 서브넷 주소를 정의하십시오.
    • checkip.amazonaws.com/을 사용하여 IP 주소를 식별하고 IP 뒤에 CIDR 서브넷 마스크 / 32를 입력하여 액세스를 구체적으로 제한할 수 있습니다(with no spaces between IP and the CIDR subnet mask).
  • InstanceType을 선택합니다. 기본 프리 티어 적격 t2.micro 옵션을 선택하는 것이 좋습니다.
  • KeyName 풀다운 목록에서 EC2 KeyPair를 선택합니다.
  • CIDR 서브넷 마스크로 IP 주소를 입력하여 SSHLocation을 정의하십시오.
  • Workload 이름을 ExampleCorp로 정의하십시오.

  1. 다음(Next)을 선택합니다.

  2. 옵션 페이지에서 선택적으로 추가 태그를 지정한 후 아래로 스크롤하여 Next를 선택합니다.

  3. Review page에서

  • 선택 사항을 검토합니다.
  • 아래로 스크롤하여 다음 문장의 체크박스의 확인란을 선택합니다.“I acknowledge that AWS CloudFormation might create IAM resources”.
  1. Create를 선택합니다.

1.2 SNS 알림 구독 확인

  • CloudFormation 스크립트가 완료되면 이전에 제공한 주소의 이메일을 확인합니다.
  • 제목이 AWS Notifications인 AWS 알림 메일을 클릭하십시오.
  • 메일 내용의 구독 확인(Confirm subscription)링크를 클릭하여 구독을 확인하십시오.

Note: 이 실습에서는 방금 만든 웹 기반 애플리케이션과 상호 작용합니다. 활동 로그를 생성하려면 애플리케이션 서버에 사진을 업로드해야합니다. 자체 이미지를 사용하거나 여기에서 사용할 샘플 이미지를 다운로드 할 수 있습니다.: Sample Photos 샘플 이미지 파일은 76MB입니다. 샘플 이미지를 사용하려는 경우 지금 다운로드를 시작하면 필요하기 전에 완료할 수 있습니다.
